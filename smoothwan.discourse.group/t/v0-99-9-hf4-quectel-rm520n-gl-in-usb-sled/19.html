I‚Äôm not aware of any default Luci benefit to one over the other. I just find the ModemManager works better with my device and hardware config. The firmware I‚Äôve used that has an interface specific to LTE modems is ROOter https://www.ofmodemsandmen.com ROOter Autobuilds GitHub - ofmodemsandmen/RooterSource: Source for ROOter Build System

In my (in)experience I‚Äôve noticed the following of ModemManager:
Pros:

  • Provides a uniform and consistent interface to manage cellular modems regardless of the underlying hardware/software config. It‚Äôs used outside the OpenWRT project.
  • Supports a wide range of modems and protocols.
  • Supports auto-connection, SIM switching, SMS, and USSD.
  • Has good integration with NetworkManager which is the default network management tool on many Linux distributions.
  • Provides reception/connection quality, diagnostics and debugging information in the cli.

Cons:

  • ModemManager can be more resource-intensive than other options.
  • Configuration and customization of ModemManager outside the basic UI provided may require more advanced technical knowledge.
